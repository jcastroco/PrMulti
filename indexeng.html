Login

The login screen allows access to the application. Below is a basic explanation of its functionality:

  • Username: Enter your username in the specified field.
  • Password: Enter your password in the corresponding field.
  • "Ok" Button: Click this button to confirm the data and proceed.
  • "Cancel" Button: Click this button to exit the application if you don't wish to continue.
Note: If the credentials are correct, access will be granted. Otherwise, you will need to verify the entered information.



User Registration

This window allows a user to register in the application by entering their details, which are saved in the database:

  • Email: Field to enter your email.
  • First Name: Field to enter your real first name.
  • Last Name: Field to enter your surname.
  • DNI: Field to enter the DNI (ID number).
  • Alias: Field to enter your alias.
  • Password: Field to enter your password.

FTP Window

Represents a graphical user interface (GUI) designed for a file management system based on FTP (File Transfer Protocol). This system enables users to manage files and directories on a remote server using the options and functionalities displayed on-screen.

Interface Description

  • "Reload" Button: Allows the user to refresh the list of files and directories displayed in the system's file tree. This ensures that any changes made, such as the creation, deletion, or modification of files, are reflected in real-time.
  • Text Box and "Rename" Button: Facilitates the renaming of a selected file or directory. The user can enter a new name in the text box, and by clicking "Rename," the system will execute the action.
  • File Tree (JTree): The file tree displays the hierarchical structure of directories and files available on the FTP server. In the example image, three main folders are shown:
    • colors
    • sports
    • food
    These folders may contain more files or subdirectories, representing the organizational system of the server.
  • Main Function Buttons: On the left side of the interface, there is a series of buttons that facilitate different file and directory management actions:
    • Create Directory: Allows creating a new directory on the FTP server.
    • Delete Directory: Facilitates the deletion of a selected directory. It is important to ensure the directory is empty before deleting it, depending on the server rules.
    • Edit Directory: Allows modifying attributes or settings of an existing directory, such as access permissions.
    • Delete File: Provides the functionality to delete a selected file from the server.
    • Download File: Enables the user to download a file from the FTP server to their local system.
  • "Exit" Button: Ends the session and closes the application.

General System Functionality

This FTP management system provides a user-friendly interface that facilitates user interaction with remote files and directories without the need to use FTP commands in a terminal. Key functionalities include:
  • Visual Navigation: Allows intuitive exploration of the FTP server's structure.
  • Directory Management: Creation, deletion, and modification of directories.
  • File Administration: Downloading, deleting, and renaming files.
  • Quick Synchronization: The "Reload" option updates the list of files and directories in real time.

Usability

The graphical interface simplifies the use of the FTP protocol, especially for users unfamiliar with command-line tools. Additionally, it provides clearly labeled buttons that allow users to perform common actions quickly and efficiently. In summary, this system automates and streamlines file and directory management tasks on an FTP server, offering a practical and visual solution for remote data management.



Email Window

The Email Window consists of two windows: The first window contains user email management,
where the following functions can be used.

  • New Email: Create a new email
  • Inbox: Access the email inbox
  • Unread: Access the list of unread emails
  • Sent: Access the list of sent emails
  • Drafts: Access the list of draft emails
  • Spam: Access the list of spam emails
  • Trash: Access the list of deleted emails

The second is the sending window,
where files can be attached to the email.
  • To: Email recipient
  • Subject: Email subject
  • Message: Email content
  • Add File: Button to attach a file to the email
  • Send: Button to send the email
  • Cancel: Button to cancel the email and return to the previous view



Configuration Window

Personal User Configuration Window: Objective: Allows a user to configure and modify their personal information within the system. Available Fields:

  • Alias: The user can set or modify their nickname or alias within the system.
  • Name: Field where the user enters their real name.
  • Last Name: Allows the user to input or update their last name(s).
  • Email: Field for the user's personal or work email address.
  • Password: Facilitates creating or changing the access password.
  • DNI: Section for the National Identity Document or a unique identification number.
  • Role Description: Field where the assigned role or responsibility of the user within the organization can be specified.
  • Available Buttons:
    • Delete: Allows the user to delete their account or remove specific data from the system.
    • Edit: Option to modify existing information fields.
    • Exit: Closes the window or returns to the main interface.

Admin Configuration Window for User Management
  • Table Structure:
    • Name: Displays the name of each user registered in the database.
    • Email: The user's email address for contact or account recovery.
    • Role: Field describing the specific role of the user within the company (e.g., worker, supervisor, manager).
  • Main Functions:
    • Create: Allows adding new users to the database.
    • Edit: Facilitates updating the existing information of a user.
    • Delete: Removes a specific user from the database.
    • Back: Returns to the main or previous interface.
